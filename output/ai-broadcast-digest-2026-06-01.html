Aaron Levie (Box CEO)

Box CEO Aaron Levie 深度分析了 企业 AI agent 的 #1 问题：上下文（context）。从 agentic coding（上下文在代码库中，用户足够技术化）到知识工作 agent，上下文问题变得更加尖锐。Box 客户每天面临的问题：知识碎片化在遗留系统中、访问控制不映射真实工作流、关键决策和流程只存在于员工脑中。他认为这是 应用 AI 公司最大的杠杆点——专注于为 agent 提供特定领域信息和专业知识，也是 FDE 和新型系统集成商的机会所在。

另一条推文中，Levie 反驳 AI 取代就业的担忧：大多数企业 CIO/CTO/CEO 对话中，AI 正在创造新岗位（FDE、工程等）或至少将效率提升节省的资金重新投入业务（销售、营销等）。引用高盛 CEO David Solomon 在纽约时报的专栏：AI 热潮既在创造全新就业岗位，也在释放资金投入此前受限的领域。

Peter Steinberger (OpenClaw Creator)

OpenClaw 创建者 Peter Steinberger 展示了多个技术亮点：

  • 用 Codex 做 QA agent：为每个 commit 自动生成用户测试场景，使用 webVNC（crabbox）和浏览器自动化（peekaboo/mcporter）像真实用户/QA 一样测试 OpenClaw，后台运行并自动提交修复 PR
  • Codex 写 ad-hoc codemods：在一次大型 TypeScript 迁移中，Codex 自动生成了临时 codemod，令他印象深刻
  • 转推 Codex 5M 用户分析：仅占 ChatGPT 约 9 亿用户的 0.6%，"我们还非常非常早期"
  • OpenClaw 理念：模块化、精简，只添加所需内容——更少的 skills 和 tools 意味着 agent 更高效
  • 签证搞定，即将搬至旧金山，赶上 MS Build 和 OpenClaw 活动
